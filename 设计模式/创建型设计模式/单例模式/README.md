# 单例模式
核心目标是确保一格类在整个程序生命周期中只有一个实例，并提供一个全局统一的访问节点来获取这个实例。
这种模式适用于需要全局唯一实例的场景，比如配置管理器，日志工具，数据库连接池，线程池等
## 单例模式在C++中怎么实现
在C++中实现单例模式，主要是确保一个类只有一个实例，并提供一个全局访问点，C++实现单例模式需要满足以下几点要求：
- 私有化构造函数：将类的构造函数设置为私有，防止外部通过new关键字创建多个实例
- 静态实例：在类的内部提供一个静态私有实例，这个实例作为整个程序的唯一实例
- 公有访问方法：提供一个公有的静态方法，通常叫 `getInstance`，用于获取类的唯一实例
- 删除拷贝构造函数和赋值操作符：为了防止拷贝或赋值来创建新的实例，需要将拷贝构造函数和赋值运算符定义为私有或者删除
单例模式有懒汉式和饿汉式两种实现方式：
- 懒汉式：类实例只有在第一次被使用时才会创建，这个时候需要注意多线程下的访问，需要利用互斥锁来加以控制，不过C++11在初始化`static`局部变量时会自动加锁（可见文件`LazySingleton.cpp`）
- 饿汉式：类实例在类被加载时就进行创建，不存在线程安全问题（可见文件`EagerSington.cpp`）
## 如何用单例模式实现一个任务队列
对于一个任务队列，对外开放的接口肯定要有`start()` `stop()` `push()` ，三个接口，任务队列的启动，关闭，加入任务，遵循生产者--消费者模型，用户来生产任务，然后任务队列启动多个线程来等待是否队列中有新任务，如果有任务，出队任务，唤醒一个线程来执行，进行消费。
