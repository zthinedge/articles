# 连接池
## 定义
MySQL连接池是**管理、复用**MySQL数据库连接的组件，目的是解决数据库连接**创建/销毁**时的性能开销问题。
<br><br>
### 管理:
    1.用maxActive(最大连接数)限制池内总连接（空闲 + 活跃）上限，防止压垮 MySQL <br>
    2.用minIdle（最小空闲数）保证池内至少有 N 个空闲连接，不足则自动新建 <br>
    3.空闲连接数超过minIdle时，触发清理逻辑，维持合理空闲量 <br>
### 复用 ：
连接不随请求销毁，循环供多请求使用

## 连接池的使用及优势
### 使用 
    1.连接池初始化
    2.业务请求获取连接
    3.业务使用连接
    4.连接归还给连接池，重置连接状态
    5.连接池后台管理
    6.连接池关闭
### 优势
    1. 提升性能
      消除连接创建 / 销毁的开销：高并发下，QPS 可提升数倍，响应时间（RT）降低 50% 以上；
      减少 MySQL 服务器的资源消耗：避免频繁创建连接导致的 CPU / 内存开销。
    2. 提高稳定性
      限制最大连接数：防止应用层无节制创建连接，导致 MySQL 崩溃；
      连接健康检查：自动剔除失效连接，避免应用访问数据库时出现 “断连” 错误；
      等待队列机制：避免瞬时高并发导致的连接直接失败，提升系统容错性。
    3. 便于监控和调优
      连接池提供丰富的监控指标：空闲连接数、活跃连接数、等待队列长度、连接超时次数等；
      可动态调整参数（如 minIdle、maxActive）：无需重启应用，即可适配业务流量变化。
    4. 资源复用
      连接归池后可被多个请求复用，提高连接的利用率，减少资源浪费。




## mysql.h里面常用的底层函数
### 1.连接管理函数
这些函数负责初始化句柄以及与 MySQL 服务器建立网络连接。
<br><br>

<u> mysql_init(MYSQL *mysql) </u>    

**作用**：分配或初始化一个 MYSQL 对象（句柄）。

**用法**：通常传入 nullptr 来让系统自动分配内存。<br><br>  


<u> mysql_real_connect()</u>

**作用**：真正建立与数据库的 TCP 连接。

**关键参数**：需要传入句柄、IP、用户名、密码、数据库名、端口等。

**返回值**：连接成功返回句柄指针，失败返回 NULL。<br><br>


<u>mysql_close(MYSQL *mysql)</u>

**作用**：关闭连接并释放句柄占用的内存。

**池化逻辑**：在连接池中，这个函数只在池子销毁时调用，平时只需保持连接开启。

### 2.SQL 执行与数据更新

mysql_query(MYSQL *mysql, const char *q)
作用：执行以 NULL 结尾的 SQL 语句。

返回值：执行成功返回 0，失败返回非零值。

mysql_affected_rows(MYSQL *mysql)

作用：返回上一次 INSERT, UPDATE 或 DELETE 语句影响的行数。

mysql_error(MYSQL *mysql)

作用：返回最近一次调用函数失败的文本描述。
### 3.结果集处理（查询必备）
mysql_store_result(MYSQL *mysql)

作用：将查询到的完整结果集一次性从服务器下载到客户端内存中。

返回：一个 MYSQL_RES 结果集指针。

mysql_fetch_row(MYSQL_RES *result)

作用：从结果集中获取下一行数据。

返回：MYSQL_ROW（本质是字符串数组）。

mysql_num_fields(MYSQL_RES *result)

作用：返回结果集中的列数。

mysql_free_result(MYSQL_RES *result)

作用：释放由 mysql_store_result 分配的结果集内存。如果不调用，会导致内存泄漏。

### 4.辅助与保活函数
mysql_ping(MYSQL *mysql)

作用：检查与服务器的连接是否仍然有效。

用法：连接池通常定期调用它来做“心跳检测”，如果连接断开，它会尝试重新连接。

mysql_set_character_set(MYSQL *mysql, const char *csname)

作用：设置当前连接的字符集（如 utf8mb4），防止中文乱码。
