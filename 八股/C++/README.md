# C++核心知识点整理

## 静态变量和全局变量，局部变量的区别，在内存上是怎么分布的
- 静态变量，如果是局部静态变量的话它的作用域就是在声明它们的函数或代码块内部，他在程序整个运行期间都存在，只初始化一次。它存储在全局存储区。
- 局部变量：它的作用域就是在声明它们的函数或代码块内部，生命周期为函数被调用时被创建，函数调用结束后销毁，同时它必须在使用前显式初始化，它存储在栈上，与它们所在的作用域相关联
- 全局变量，在整个程序中都是可见的，可以在任何函数或代码块中访问，它的生命周期为程序的整个运行期间，通常在程序启动时初始化，也存储在全局存储区

## 指针和引用的区别
- 指针是一个存储地址的变量，它指向内存中的一个位置，引用就是一个变量的别名
- 指针可用被初始化为空，表示不指向任何地址，引用在定义时必须被初始化，不能引用空的内存地址
- 指针可用改变指针的指向，使其指向不同的内存地址，引用一旦被初始化，就会一直指向一个对象，不能改变绑定
- 指针可以通过解引用来获取指针指向变量的值，还可以通过取地址运算符获取变量的地址；引用在声明时被初始化，在整个生命周期中一直引用同一个变量，引用本身就是变量的别名
- 指针通常用于动态内存的分配，数组操作以及函数参数传递；引用通常用于函数参数传递、操作符重载和创建别名

## C++的内存分区
c++的内存分区一般分为五个区域：
- 栈：用于存储局部变量和函数调用的上下文。栈的内存分配是自动的，由编译器管理
- 堆：用于动态内存分配。可以用new、malloc从堆上分配内存，并用delete、free释放内存
- 全局/静态存储区：存储全局和静态变量。
- 常量存储区：用来存储程序中的常量数据
- 代码段：存储程序的可执行代码和函数的二进制命令


## static关键字和const关键字的作用
static和const是c++中两个常用的关键字
其中static关键字用于控制变量和函数的生命周期、作用域和访问权限
- 声明静态变量：静态变量的生命周期直到程序结束。当在函数内部声明静态变量时，即使函数执行完了也不会释放它，下次调用该函数时会保留上次的状态
- 在类中，被static声明的成员被称为静态成员 ：如静态成员变量，在类中使用static关键字修饰的成员变量，表示该变量属于类而不是类的实例，所有实例共享同一份数据。静态成员函数，在类中使用static关键字修饰的成员函数，所有对象共享一个函数。静态成员函数只能访问静态成员变量，静态成员函数的调用可以不通过创建类的实例，而是直接通过类名调用
- static变量如果被多个线程访问，要注意线程安全的问题

const关键字用于定义常量，一旦初始化后就不能被修改
- 常量变量：声明变量，使变量只读
- 常量成员函数：表示该函数不会修改对象的成员变量
- 常量指针：指向一个const的值，或者指向一个const值的指针，表明指针指向的值不能通过这个指针被修改
- const由于不可变性，天然具有线程安全性

## 常量指针和指针常量有什么区别
它们主要的区别在于它们所指向的数据是否可以被修改，以及它们的值是否可以被修改。
- 常量指针是指指针所指向的数据是常量，不能通过这个指针来修改他所指向的数据。但是指针本身的值是可以修改的。
- 指针常量是指指针本身的值是常量，一但初始化后就不能指向其他地址。但是它所指向的数据是可以修改的（除非那个数据本身是常量）

## 结构体和类的区别
- 结构体中的成员默认是public的，class中的成员默认是private的
- class继承默认是private，struct默认继承是public
- class可以用于定义模板函数，struct不行

## 什么是智能指针，C++有哪几种智能指针？
智能指针是C++中用来自动管理动态分配内存的一种模板类，维护着一个指向动态分配对象的指针，并在智能指针对象被销毁时，自动释放内存，从而避免内存泄漏。

C++有三种智能指针，分别是：
- `unique_ptr`：独占所有权，保证同一时间只有一个unique_ptr指向特定内存，适用于不需要共享所有权的场景，比如栈上对象的管理
- `shared_ptr`：多个shared_ptr共享同一内存，使用引用计数机制来管理内存的生命周期，适用于多个对象需要共享同一资源的场景
- `weak_ptr`：弱引用，用来解决shared_ptr可能导致的循环引用问题，它不拥有所指向的对象

## 智能指针的实现原理是什么
1. `unique_ptr`：
- unique_ptr代表独占所有权的智能指针，同一时间只能有一个unique_ptr实例指向特定资源；
- 它通过析构函数释放资源，当unique_ptr超出作用域时，会自动调用删除操作符来释放其指向的内存。
- 它所有权的唯一性是通过删除复制构造函数和复制赋值运算符来完成的，不过它提供移动构造函数和移动赋值运算符，允许所有权的转移
2. `shared_ptr`：
- shared_ptr允许多个智能指针的实例共享同一资源的所有权，使用引用计数机制来跟踪有多少个shared_ptr指向同一资源；
- 它内部维护了一个控制块，通常包含引用计数和资源的原始指针。每创建一个新的shared_ptr或者将一个shared_ptr赋值给另一个时，引用计数增加。
- 当shared_ptr被销毁或者通过标准库提供的reset成员函数重置时，引用计数减少。当引用计数降到0的时候，控制块会释放资源并自我销毁
3. `weak_ptr`:
- weak_ptr是一直不拥有对象的智能指针，它观察shared_ptr管理的对象，但不增加引用计数
- 它主要是用来解决shared_ptr之间可能存在的循环引用问题，因为循环引用会导致引用计数永远不能归零，从而导致内存泄漏
## new和malloc有什么区别
new和malloc都用于动态内存分配，有以下几个关键区别：
1. 语法层面
- new是C++的操作符，可以直接用来分配对象或数组
- malloc是一个函数，通常需要包含头文件<cstdlib>，并且只分配原始内存
2. 类型安全
- new是类型安全的，会根据分配的对象类型进行正确的内存分配和构造函数调用
- malloc不是类型安全的，只分配原始内存，不调用构造函数。返回类型是void*，需要强制转换成具体的指针类型，但是强转时编译器不会报错，但是可能由于在四字节的内存里塞了八字节的数据（void*->double*）运行的时候就会崩溃
3. 构造和析构
- 使用new分配的对象在对象生命周期结束时需要使用delete来进行释放，delete会调用对象的析构函数
- 使用malloc分配的内存需要用free来释放，free不会自动调用析构函数，因此如果分配的是对象数组，需要手动调用析构函数。或者在构造时是先malloc分配内存，再在内存上new对象
4. 异常安全性
- new在分配失败时会抛出bad_alloc异常
- malloc在分配失败时会返回NULL指针
5. 管理机制C++中的new和delete通常由编译器实现，可能包含一些额外的内存管理机制
- C语言的malloc和free是由C标准库提供的，与编译器无关

总的来说，new和malloc都是动态内存的分配手段，但是new提供了类型安全和构造/析构的自动化，而malloc提供了更底层的分配方式，需要手动管理构造和析构。

## delete和free有什么区别
delete和free都是用来释放动态分配的内存，区别如下：
1. 语法
- delete是C++的关键字，用来释放new分配的对象
- free是C语言中的函数，包含在<stdlib.h>头文件中，用于释放由malloc分配的内存
2. 对象销毁
- 当使用delete释放对象内存的时候，C++编译器会自动调用对应的析构函数，释放与对象相关的资源，并执行对象的清理工作
- free仅释放内存，不调用析构函数。因此如果使用malloc分配了C++对象的内存，需要手动调用析构函数后再调用free。
3. 数组处理
- 如果是数组，C++提供了delete []来释放整个数组的内存，而C语言中仍然使用free，没有区分单个对象和数组
4. 返回值
- free没有返回值，即使内存释放失败也不会有任何反馈信息
- delete之后，指针依然保留原来的地址，这个状态被称为悬空指针（堆上内存已释放，指针还在栈上），在释放指针后，立刻手动置为NULL,避免出现悬空指针导致访问已释放的内存
5. 类型检查
- delete会进行类型检查，确保删除的对象和new分配的对象类型一致
- free不进行类型检查，因为它只处理void*类型的指针，任何类型的指针都可以隐式转换为void*

总的来说，delete和free都是用来释放动态内存的，但它们分别用于C++和C语言中的内存管理。delete适用于C++对象，会自动调用析构函数，而free适用于C语言分配的内存，不涉及对象的析构。

## 什么是内存泄漏？如何检测和防止
1. 如果程序的某一段代码在内存池中动态申请了一块内存而没有及时将它释放，就会导致那一块内存一直处于被占用的状态而无法使用，造成了资源的浪费。内存泄漏并不是说物理上的消失了，是因为无法使用该区域，在外界看起来内存好像被泄漏了一样。
2. 造成内存泄漏的主要有以下几种情况：
- 忘记释放内存：使用new 或malloc等分配内存之后，没有使用delete或free来进行对应内存的释放
- 循环引用：在使用shared_ptr时，循环引用会导致引用计数永远不会归零，导致内存无法释放。
- 使用不匹配的内存释放函数：比如使用delete释放new []分配的内存，或者使用delete []释放new分配的内存，这可能会导致未定义行为。
- 指针被赋值未空或者重新赋值后，丢失了对原有堆对象的引用，导致内存无法释放，造成内存泄漏
- 子类继承父类时，没有将基类的析构函数定义为虚函数，导致子类的资源不会析构。因为非虚析构，在delete基类指针的时候，编译器静态绑定，只认指针的类型调用基类的析构；虚析构，在delete时，编译器动态绑定，认指针实际指向的对象类型，先调用子类的析构函数，再调用基类的析构函数，这就是析构函数的多态。
- 资源未关闭：对于文件、网络连接等资源，如果没有正确关闭，虽然不直接导致内存泄漏，但会占用系统资源，可能导致资源耗尽。
3. 如何检测：在Linux下一般使用Valgrind、AddressSanitizer进行检测，Windows下VS的debug模式可以自动检测内存释放泄漏
4. 如何避免
- 使用智能指针，因为智能指针支持自动管理内存
- 确保资源释放，对于手动分配的内存，确保在不再需要使用时用delete或free进行释放
- 内存泄漏检测工具：在开发和测试阶段，定期使用内存泄漏工具检查程序

## 什么是野指针，如何避免
1. 什么是野指针：
野指针是指指向已经被释放或者无效的内存地址的指针。在C和C++这类直接操作内存地址的语言中，如果指针没有被正确初始化，或者指针所指向的内存以及被释放，那么这个指针就成了野指针。使用野指针可能会导致程序崩溃、数据损坏或者其他一些不可预测行为
2. 在什么情况下会产生野指针/
- 在释放后没有置空指针：使用delete或free释放内存后，没有将指针设置为nullptr，指针仍然指向已释放的内存地址
- 返回局部变量的指针：如果函数返回了其指向局部变量的指针，一旦函数返回，这些局部变量的生命周期结束，返回的指针就成为了野指针
- 越界访问：指针访问的内存超出了其合法的内存块边界
- 函数参数指针被释放，导致指向的内存被释放，外部的指针指向的内存无效了
3. 如何避免野指针
- 在释放内存时将指针置空
- 避免返回局部变量的指针
- 使用智能指针
- 注意函数参数的生命周期，避免在函数内释放调用方的指针。可以通过引用传递指针。

## 堆和栈的区别
堆和栈是程序运行时两种不同的内存分配区域
1. 内存分配
- 栈：是由编译器自动管理的，用于存储局部变量和函数调用的上下文信息；栈上的对象在定义它们的块或者函数调用结束后自动销毁；栈的内存分配和释放速度很快，因为栈的内存管理是连续的，分配内存只需要修改一个栈指针，不需要搜索空闲内存
- 堆：是由程序员手动管理的，用于存储动态分配的对象；堆上的对象需要由程序员手动释放，否则可能会造成内存泄漏；堆上的内存分配和释放通常比栈慢，因为需要搜索到合适的内存块，而且堆管理器还需要整理内存碎片
2. 大小限制
- 栈的内存大小是固定的，在windows下默认的内存大小为1MB，Linux和mac下默认内存大小为8MB
- 堆内存大小几乎无上限，只受限于系统可用内存大小
3. 使用场景
- 栈主要用于存储函数参数，局部变量等
- 堆用于存储生命周期不受函数生命周期限制的对象，比如用new malloc分配的对象

## C++面向对象三大特性
面向对象编程是 C++ 的核心特性之一。OOP 是一种编程范式，通过它，程序可以以对象的形式组织数据和功能，并通过对象之间的交互来实现任务。面向对象编程具有封装、继承和多态三个主要特性：
1. 封装：将客观事物封装成为抽象的类，类把自己数据和方法进行隐藏，仅对外公开接口来和对象进行交互，防止外界干扰或不确定性访问。
2. 继承：指一个子类可以从父类中继承属性和行为的能力。通过继承，子类可以重用父类的代码，并且可以在不修改父类的情况下添加新的功能或修改已有的功能。继承使得代码具有层次性和可扩展性，能够建立起类与类之间的层次关系。
3. 多态：多态是指同一个操作作用于不同的对象时，可以有不同的解释和行为。多态性允许以统一的方式处理不同类型的对象，从而提高了代码的可扩展性和可维护性。在 C++ 中，多态性通常通过虚函数来实现。通过基类中定义虚函数，并在派生类中重新定义该函数，可以实现运行时的动态绑定。
