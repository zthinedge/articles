# MySQL 连接池技术手册

## 1. 定义
MySQL 连接池是**管理、复用** MySQL 数据库连接的组件。其核心目的是解决数据库连接在频繁**创建与销毁**过程中产生的性能开销问题。

---

## 2. 核心机制

### 🛠️ 管理 (Management)
1. **最大活跃数 (`maxActive`)**：限制池内总连接（空闲 + 活跃）的上限，防止应用端因无节制申请连接而压垮 MySQL 服务器。
2. **最小空闲数 (`minIdle`)**：保证池内至少预留 N 个空闲连接。当空闲连接不足时，连接池会自动新建连接以备不时之需。
3. **动态清理**：当空闲连接数超过 `minIdle` 且闲置时间过长时，触发清理逻辑，维持合理的空闲资源量。

### 🔄 复用 (Reuse)
* **长连接机制**：连接不随单次请求的结束而销毁，而是重新回到池中循环使用，供多个请求轮番调度。

---

## 3. 工作流程与优势

### 📑 运行步骤
1. **连接池初始化**：程序启动时预创建基础连接。
2. **获取连接**：业务请求从池中“借用”现成的连接。
3. **执行业务**：使用连接进行 SQL 操作。
4. **归还连接**：重置连接状态并回池（并非物理关闭）。
5. **后台管理**：定期进行健康检查（如 `mysql_ping`）和数量调优。
6. **连接池关闭**：程序退出时安全释放所有资源。



---

### 🚀 核心优势

| 优势维度 | 详细描述 | 预期效果 |
| :--- | :--- | :--- |
| **提升性能** | 消除 TCP 三次握手及认证开销。 | QPS 提升数倍，响应时间 (RT) 降低 50% 以上。 |
| **提高稳定性** | 限制最大连接数，提供等待队列。 | 防止 MySQL 崩溃，自动剔除失效的“死连接”。 |
| **便于调优** | 提供空闲/活跃数等实时监控指标。 | 支持动态调整参数以适配业务流量波动。 |
| **资源复用** | 提高连接利用率，减少资源浪费。 | 显著降低服务器 CPU 和内存的无效占用。 |

## mysql.h里面常用的底层函数

### 1. 连接管理函数 
这些函数负责初始化句柄以及与 MySQL 服务器建立网络连接。

* **`mysql_init(MYSQL *mysql)`**
    * **作用**：分配或初始化一个 `MYSQL` 对象（句柄）。
    * **用法**：通常传入 `nullptr` 让系统自动分配内存。
* **`mysql_real_connect(...)`**
    * **作用**：建立与数据库的 TCP 连接。
    * **参数**：句柄、IP、用户名、密码、数据库名、端口等。
    * **返回值**：成功返回句柄指针，失败返回 `NULL`。
* **`mysql_close(MYSQL *mysql)`**
    * **作用**：关闭连接并释放句柄占用的内存。
    * **池化逻辑**：在连接池中，此函数仅在池子销毁时调用，平时保持连接开启。

---

### 2. SQL 执行与数据更新 
用于向服务器发送指令，通常对应增、删、改（DML）操作。

* **`mysql_query(MYSQL *mysql, const char *q)`**
    * **作用**：执行以 `NULL` 结尾的 SQL 语句。
    * **返回值**：执行成功返回 `0`，失败返回非零值。
* **`mysql_affected_rows(MYSQL *mysql)`**
    * **作用**：返回上一次 `INSERT`, `UPDATE` 或 `DELETE` 语句影响的行数。
* **`mysql_error(MYSQL *mysql)`**
    * **作用**：返回最近一次调用函数失败的文字描述，常用于日志排查。

---

### 3. 结果集处理
执行 `SELECT` 语句后，必须通过以下函数解析数据。



* **`mysql_store_result(MYSQL *mysql)`**
    * **作用**：将查询到的完整结果集一次性从服务器下载到客户端内存。
    * **返回**：一个 `MYSQL_RES` 结果集指针。
* **`mysql_fetch_row(MYSQL_RES *result)`**
    * **作用**：从结果集中获取下一行数据。
    * **返回**：`MYSQL_ROW`（本质是字符串数组）。
* **`mysql_num_fields(MYSQL_RES *result)`**
    * **作用**：返回结果集中的列数。
* **`mysql_free_result(MYSQL_RES *result)`**
    * **作用**：**必须调用**。释放结果集占用的内存，否则会导致内存泄漏。

---

### 4. 辅助与保活函数 

* **`mysql_ping(MYSQL *mysql)`**
    * **作用**：心跳检测。检查与服务器的连接是否有效。
    * **用法**：若断开，它会尝试根据配置自动重连。
* **`mysql_set_character_set(MYSQL *mysql, const char *csname)`**
    * **作用**：设置字符集（如 `utf8mb4`），解决中文乱码问题。

---

> **💡 开发建议：**
> 在 C++ 中使用时，建议将 `MYSQL_RES` 封装在 **RAII 对象**中，确保在任何情况下（如抛出异常）都能自动执行 `mysql_free_result`。
