# 内存池（Fixed-Size Memory Pool）使用指南

本项目实现了一个**固定大小的内存池**，用于高效管理固定大小内存块的分配与释放。适用于需要频繁创建/销毁相同大小对象的场景（如链表节点、消息结构体、游戏中的粒子等），可以显著减少 `malloc/free` 的开销和内存碎片。


## 为什么使用内存池

- **性能提升**：分配/释放时间接近 O(1)，远快于 `malloc/free`。
- **减少碎片**：所有块大小固定，内部管理避免外部碎片。
- **复用机制**：释放的块会立即放回空闲链表，下次分配优先复用（不会重复向系统申请内存）。
- **可预测性**：避免系统分配失败或突发延迟。

**适用场景**：
- 高频创建/销毁小对象（如链表节点、消息包）。
- 游戏引擎、服务器、网络程序等对性能敏感的系统。

**不适用场景**：
- 需要分配不同大小的对象（可结合多层池解决）。
- 只偶尔分配内存（直接用 `new/malloc` 更简单）。
